# Related Work Revision Guide

## 1. Purpose

Related Work must serve contribution positioning. It should not be a literature dump.

The reviewer should understand:
1. Why this is not text watermarking.
2. Why this is not generic trigger-response fingerprinting.
3. Why clean success parity with Perinucleus does not eliminate the paper’s contribution.
4. Why RS decoding is not claimed as new coding theory.
5. What robustness regimes are covered by related work but not by this paper.

## 3. Recommended Related Work Structure

### Category 1: Text Provenance and Watermarking

Purpose:
- Explain that text provenance asks whether text was generated by a marked model or rule.
- It does not recover an owner-controlled payload from a suspect model.

Required comparison:
- KGW-style token watermarking.
- Unbiased/distortion-free watermarking.
- Robust watermarking.
- Watermark removal.
- Radioactive watermarking if present.

How to position our work:

```latex
Text watermarking and provenance methods ask whether a text sample carries a generation signal. Our verifier asks a different question: whether a suspect service emits owner-controlled evidence that decodes to a committed payload under a public rule. This distinction changes both the evidence object and the evaluation metrics: payload recovery and false-accept calibration matter in addition to text-level detectability.
```

### Category 2: Active LLM Fingerprinting and Ownership Verification

Purpose:
- This is the closest category.
- Must explain why this paper is not merely another trigger-response fingerprint.

Required comparison:
- Instructional fingerprinting.
- HuRef.
- Chain & Hash.
- LLMmap.
- Scalable/Perinucleus.
- ROFL.
- ESF/RESF or other tamper-detection fingerprints if present.
- CTCC / EverTracer if present.

How to position our work:

```latex
Active LLM fingerprinting is the closest comparison family. These methods implant or discover model-side behaviors that identify a suspect model under selected queries. Our distinction is not higher clean success in the current Qwen comparison: the matched clean matrix is parity with the Qwen-adapted Scalable/Perinucleus baseline. The distinction is structural. We define the evidence event so that it is first-token measurable, train the verifier-relevant bucket mass, and use a public decoder to recover a payload rather than relying only on an implicit response pattern or learned identification rule.
```

```latex
\textbf{TODO_AFTER_RESULTS:} Insert matched clean comparison with the Qwen-adapted Scalable/Perinucleus baseline. Do not claim clean superiority unless the final artifact supports it.
```

### Category 3: Robustness Under Modification, Erasure, Merging, and Scrubbing

Purpose:
- Explain that other work studies robustness to model merging, erasure, backdoor removal, or persistent behaviors.
- This paper should not claim broad robustness unless experiments support it.

Required comparison:
- MergePrint.
- Meraser / fingerprint erasure.
- Backdoor removal.
- LoRA / PEFT scrubbing.
- Model soups, task arithmetic, TIES-Merging.
- Persistent backdoors / sleeper agents if used as motivation.

How to position our work:

```latex
Post-release modification work defines adversarial transformations that any ownership method must either evaluate or exclude. Our verifier provides a conditional decoding statement: bucket-preserving variation is absorbed and residual parseable corruption is handled as symbol errors and erasures. It does not claim robustness to carrier-block deletion, delimiter destruction, or probe-free distillation.
```

### Category 4: Classical DNN Ownership Proofs

Purpose:
- Provide historical context.
- Explain why autoregressive LLMs introduce a tokenizer/next-token interface problem.

Required comparison:
- Parameter watermarking.
- Backdoor trigger ownership.
- DeepSigns.
- Entangled Watermarks.
- DAWN.
- ClearStamp.

How to position our work:

```latex
Classical DNN ownership proofs establish precedents for parameter signatures, trigger sets, and activation-level ownership evidence. In autoregressive LLM services, however, the owner observes generated text rather than parameters or activations. This makes tokenizer alignment, next-token measurability, and public text decoding central design constraints.
```

### Category 5: Passive Provenance, Extraction, Membership, and Imprint

Purpose:
- Explain complementarity.
- These methods infer lineage or exposure without active payload injection.

Required comparison:
- Production model extraction.
- Model extraction revisited.
- Membership inference.
- Training data imprint.

How to position our work:

```latex
Passive provenance and extraction analyses can support forensic reasoning about model lineage or training exposure. They do not replace an active audit protocol in which an owner commits to a payload and a public verifier attempts to recover it from suspect outputs.
```

### Category 6: Coding Layer

Purpose:
- Prevent reviewer from thinking RS coding is claimed as novel.

Required comparison:
- Reed--Solomon classical code.

How to position our work:

```latex
We use Reed--Solomon decoding as a standard coding primitive. The contribution is the reduction from bucketized text evidence to a bounded symbol error/erasure model, not a new coding theorem.
```

## 4. Recommended Main Related Work Opening

Replace a generic related work opening with:

```latex
Prior work differs along three axes that determine whether it addresses our problem: the evidence object, the trainability interface, and the verification rule. Text watermarking verifies generated-text provenance; active fingerprinting verifies model-associated behavior; classical DNN watermarking often relies on parameters, activations, or trigger responses; passive provenance infers exposure or lineage. Our setting requires a recoverable owner payload whose event is exactly trainable at the next-token interface and whose recovery is checked by a public decoder.
```

## 7. What Not to Do

Do not:
- List 20 papers without explaining the axis of difference.
- Claim our method is more robust than MergePrint unless experiments support it.
- Treat KGW/PostMark as primary ownership baselines.
- Hide Perinucleus clean parity.
- Claim related work lacks tokenizer awareness unless verified.
- Invent citation keys.
- Add citations without bibliography entries.
- Overstate that no prior work considers payloads; use careful language such as “not the central evidence object” if uncertain.

## 9. Reviewer Concern Served

This revision serves these reviewer concerns:

- Originality: clarifies what is new.
- Significance: explains why next-token measurability matters.
- Quality: avoids unfair baseline positioning.
- Clarity: prevents literature dump.
- Scope control: distinguishes robustness regimes.
